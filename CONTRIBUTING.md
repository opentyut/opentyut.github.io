# 贡献指南

感谢你对 OpenTYUT 的关注！这份指南帮助你高效地为项目做出贡献。

## 行为准则

OpenTYUT 是一个面向新手的友好社区。在任何互动中，请保持尊重、耐心和建设性。

- **Be nice.** 每个人都是从新手过来的。
- **Be helpful.** 如果看到你能回答的问题，欢迎帮忙。
- **Be constructive.** 批评可以，但要给出改进建议。

## 我可以贡献什么

| 我擅长 | 我可以 |
|--------|--------|
| 某门课程学得好 | 补充课程笔记、学习心得、考试复习要点 |
| 用过好用的工具/网站 | 推荐到"外部资源"页面 |
| 发现网站有错误 | 提 Issue 或直接 PR 修正 |
| 写前端 | 改进网站样式、功能、交互 |
| 什么都不会但想帮忙 | 检查死链、修正错别字、转发给需要的同学 |

## Issue 规范

### 提 Issue 之前

1. 搜索已有 Issues，确认不重复
2. 阅读 [提问的艺术](https://opentyut.github.io/guide/how-to-ask)
3. 选择合适的 Issue 模板

### Issue 不会被回复的情况

- 问题描述只有一句话，没有上下文
- 明显可以在网站或搜索引擎找到答案的问题
- 态度恶劣或不尊重他人

## Pull Request 规范

### 分支命名

```
feat/xxx       新增内容或功能
fix/xxx        修复 bug 或错误
docs/xxx       文档相关
chore/xxx      杂项（配置、依赖等）
```

### Commit 信息

采用 [Conventional Commits](https://www.conventionalcommits.org/zh-hans/)：

```
feat: 新增大一上学期 C 语言笔记
fix: 修正 Git 指南中的死链
docs: 更新 README 贡献说明
```

### PR 流程

1. Fork 本仓库
2. 创建你的分支
3. 提交修改（遵循 commit 规范）
4. 推送并创建 PR
5. 等待 Review

### PR 合并标准

维护者会检查以下事项：

- [ ] 内容准确、来源可靠
- [ ] 无敏感信息（个人信息、密码、密钥等）
- [ ] 排版清晰、链接有效
- [ ] 未引入大量无关变更

## 内容规范

### Markdown 排版

- 标题层级合理，不跳级
- 代码块标注语言类型
- 外链使用完整 URL
- 图片使用相对路径，放在 `public/` 目录下

### 信息可靠性

- 课程相关信息请确认与当前教学计划一致
- 外链资源请确认链接有效、内容安全
- 个人经验建议注明适用前提（"我当时……"、"某级某专业……"）

## 本地开发

```bash
npm install
npm run dev
```

访问 `http://localhost:5173` 预览。

不需要前端知识即可参与内容贡献——直接在 GitHub 网页上编辑 Markdown 文件即可。
